1. 引言

近年来涌现出一批自动化研究系统,包括 Auto Research [1]、 AI Scientist [2] 以及 Analemma 的 FARS [3]。 这些工作在端到端自动化科学研究方面取得了令人印象深刻的进展。 与其提出又一个复杂的框架,我们研究一个更直接的问题: 广泛使用的通用 CLI 智能体(例如 Claude Code、Codex、Kimi Code)能否在极少指导下完成研究。 此外,目前仍缺乏针对智能体研究在不同领域、不同算力条件(如仅 CPU 与 GPU 环境)下的系统性质量评估,难以真正衡量其在现实研究场景中的能力与局限。

We evaluate three off-the-shelf CLI agents — Claude Code with Opus 4.6 [4, 5], Codex with GPT-5.4 [6, 7], and Kimi Code with K2.5 [8, 9] — using a minimal scaffold on 13 diverse CS domains (5 CPU-only + 8 GPU environments). We employ a peer-review protocol where three CLI agents evaluate each paper alongside its code, and additionally use the Stanford Agentic Reviewer [10] for external validation.

关键发现

每个 CLI 智能体发展出了明显不同的研究风格:Claude Code 是全栈研究者——在两种策略间保持平衡(46% 实证研究、33% 新方法),产出的论文最长、也最全面。Codex 是实证科学家——87% 是实证研究,诚信度最高,但新颖性较低。Kimi Code 是系统搭建者——其 79% 的论文都以新方法、缩写丰富的命名呈现,但新颖性得分依然较低,因为它的许多想法更像是对已有工作的重新包装,而非真正新颖的贡献。

斯坦福智能体评审 (SAR)(0–10,ICLR 量表):

  • 在自动化系统中:Claude Code (5.45) > FARS (5.06) > Codex (4.93) > Kimi Code (4.24)。我们只使用 $200 Max 订阅的 Claude Code 便超过了预算高达 $186,000 的 Analemma FARS (5.06)。
  • 与 200 篇人类撰写的 ICLR 2025 论文相比,Claude Code (5.45) 高于按录用率加权后的人类投稿均值 (5.42),介于被接收 (5.59) 与被拒绝 (5.34) 之间。

同行评审 (PR)(0–10,ICLR 量表):

  • 在同行评审中,Claude Code (4.59) > Codex (4.51) > Kimi Code (3.38)。我们的同行评审比 SAR 更严格,因为评审者可以访问实验代码和执行日志,能系统地检查是否存在捏造或缺乏支持的结果——这些问题有时连人类专家也可能忽略。
  • CLI 智能体的主要弱点是实验严谨性与重要性。它们在引用完整性(Codex: 8.41/10)和清晰度(Codex: 7.47/10)上表现相对较好。
  • 各领域之间表现差异显著——Claude Code 从 5.33(概率方法)到 3.78(机器学习隐私)不等。使用更强的 GPU(RTX A6000→H100)整体上并未带来提升,某些情形下反而分数更低,这进一步印证了上述主要弱点。
  • 通过人工检查我们进一步确认:实验严谨性是首要弱点——智能体在实验的规划、撰写与执行上都存在明显不足,限制了论文的重要性与覆盖范围。捏造结果、偏好报告负结果是智能体评审者自身评审质量不足的原因,由此引出对前沿模型忠实性的担忧。总体而言,虽有少数论文提供了有价值的洞见,但与顶级会议的水准仍有很大差距——距离真正的自动化研究仍有很长的路要走。

2. 框架概述

Each CLI agent follows a standardized pipeline:

  1. Ideation — Generate a research idea and experiment plan; self-review for up to 3 iterations.
  2. Experiments — Write and execute code, collect results; self-review for up to 3 iterations.
  3. Paper Writing — Produce a paper; self-review for up to 3 iterations.
  4. Review — Evaluate via Stanford Agentic Reviewer and triple peer review (all three agents review each paper alongside its code).

Hardware: CPU-only or 1× NVIDIA RTX A6000 (48GB). Each agent gets 4 CPUs, 60GB RAM. We also conduct experiments for GPU experiments using NVIDIA H100 (80GB) GPUs.

Peer review (PR): Every paper is reviewed by all three agents (Claude Code, Codex, Kimi Code), scoring on 9 dimensions (novelty, soundness, significance, clarity, reproducibility, experimental rigor, references, reference integrity, results integrity) on a 0–10 ICLR scale.

Stanford Agentic review (SAR): Every paper is reviewed by Stanford Agentic Reviewer, which provides ICLR-calibrated scores on a 0–10 scale.

诚信与引用分析

由于我们的同行评审可以访问实验代码与日志,评审者能够发现 SAR 无法发现的诚信问题。我们(作者)就四类诚信问题对每篇论文进行了人工标注:

  • 结果不一致 — numbers reported in the paper don't match the actual results.json, logs, or experiment outputs (e.g., Table 2 reports 0.85 accuracy but stored output shows 0.78).
  • 设置不一致 — the paper claims to do things the code doesn't actually do: claimed components or ablations are never implemented, hyperparameters in the text differ from the config, or the method is not implemented as described (e.g., the paper describes a three-stage pipeline but the code only runs two).
  • 两者兼有——同时存在结果不一致与设置不一致的论文。
  • 虚假引用 — citations that don't exist, have fabricated authors, or have incorrect bibliographic metadata.

6. 研究风格分析

10. 案例研究:论文与产物的不一致

本节我们呈现六个典型案例研究,用于支撑人工检查得出的结论。每个案例都反映了我们在 117 篇智能体论文中观察到的一种特定的失败模式,覆盖全部三个智能体,并嵌入了论文 PDF 供读者直接查阅。

案例 1 >Case 1 · Claude Code

When Do Causal Discovery Algorithms Disagree? Diagnosing Assumption Violations via Per-Edge Profiling

Claude Code 偶尔能产出带有真正洞见的论文,但较弱的实验证据仍使其整体不够有说服力。

该论文提供了一个真正有用的观察:分布型诊断能被较可靠地识别出来,而结构型诊断则仍接近随机水平。这种非对称性是一个有意义的发现,说明智能体生成的论文仍能揭示一些非平凡的经验性洞见。但整体上论文的说服力仍然不足,因为证据较弱:结果并不强,实验设置在代码产物中显得混乱,论文有时还在并非最优或次优的情形下着重强调自身方法。此外,一些关键概念的定义并不清晰,这进一步削弱了论文的忠实性。

案例 2 >Case 2 · Claude Code

The Algebra of Compiler Passes: An Empirical Study of Idempotency, Commutativity, and Convergence in LLVM Optimization Pipelines

当实验较弱时,Claude Code 可能夸大其词或呈现缺乏支持的结果。

该论文声称在 87 个基准上进行评估,但代码产物实际上只覆盖了 20 个基准的子集。论文还存在引用错误,并大量依赖合成程序,因此在规模和实际价值上都存在夸大。该案例印证了我们的观察:当实验无法提供足够有力的证据时,智能体可能通过夸大论断或呈现缺乏支持的结果来加以掩盖。它也说明了考虑代码产物的评审至关重要——仅看论文很难察觉这种不一致,只有检查代码与输出后问题才变得明显。

案例 3 >Case 3 · Codex

Do Corruption-Family Text Residuals Help Zero-Shot CLIP? A Controlled Baseline Study

Codex 在一定程度上通过缩小实验范围来降低捏造风险。

Codex 通常不追求大规模或野心勃勃的评估,而是进行受控但非常有限的实验。本案例中只使用了一个冻结的 CLIP 主干在 CIFAR-10 上进行评测,经验覆盖范围过窄,难以支撑更广泛的结论。其核心思想也与已有的 prompt-based 及无标注自适应方法十分接近,新颖性较有限。该案例印证了我们的观点:Codex 较低的捏造率部分源自其更加保守的实验方式,但这种保守也有代价——证据过于有限,使得论文整体较弱。

案例 4 >Case 4 · Kimi Code

DU-VPT: Decomposed Uncertainty-Guided Visual Prompt Tuning for Test-Time Adaptation

Kimi Code 经常直接捏造实验结果,而不是真正运行实验。

代码产物中包含硬编码的基准统计量,所谓的每次运行指标实际上是在这些常数附近采样得到的,而并非来自真实模型输出。论文中公布的结果几乎与这些预先写好的目标值完全吻合。此外,论文中声称的若干分析,包括遗忘分析和 shift 类型诊断的准确率,在代码产物中既没有对应实现也没有日志。该案例印证了我们的结论:Kimi Code 往往是在直接捏造结果,而不是通过真正的实验得到它们。

案例 5 >Case 5 · Kimi Code

UniSched: A Critical Analysis of Simulation-Based Evaluation for CXL-Aware CPU Scheduling

即便 Kimi Code 产出了代码,其方法与实现也经常对不上。

论文中报告的结果和设置与代码产物不一致,代码本身也存在明显问题,包括 PMU 任务分类的实现漏洞。结果文件显示的行为同样与论文的论断相悖,例如在方法叙述暗示迁移次数应为零的配置下,结果文件中却出现了非零的迁移计数。与案例 4(主要问题是直接捏造)不同,本案例表明即使代码存在,实现的系统也常常与论文所描述的方法对不上。由此可见,Kimi 的失败不仅仅是虚假的数字,还包括方法、代码与评估之间更深层的不一致。

案例 6 >Case 6 · Claude Code

Characterizing Operator Interaction Effects in Data Cleaning Pipelines

常见的一种失败是即使工作与已有研究显著重合,论文仍遗漏了最相关的基线。

该案例揭示了智能体论文中的一种常见失败:即使所提出的研究与既有工作有显著重合,论文仍然遗漏了最相关的先前基线。本案例中的论文与 ShapleyPipe 高度相似,却没有引用或与其进行比较。因此,评估在核心层面并不完整——缺少了最相关的基线,论文既无法令人信服地证明新颖性,也无法证明实证优势。该案例支持我们更广义的观察:许多智能体论文主要与更旧或更易击败的基线进行比较,而忽略了最近、也最相关的方法。

11. 人工检查:发现与启示

对全部 117 篇智能体论文、其代码产物及 351 份同行评审进行人工检查后的综合性总结。

论文呈现出什么样的特征

我们人工检查了所有论文及其对应的代码产物与评审,识别出若干反复出现的规律。我们将论文分为三类:方法类论文、实证研究类论文,以及基准(benchmark)类论文。在这三类中,方法类与基准类论文多为已有工作的启发式或增量式扩展,而实证研究类论文在深度上普遍有限。

各智能体在哪里失败

我们发现,所有智能体的主要弱点都是实验设计与严谨性不足。

  • Claude Code 做的实验数量最多,但评估范围依然较窄,常常缺少与最新基线的比较(见案例 6)。此外,当实验失败或结果不理想时,它有时会表现出捏造结果的迹象(见案例 2)。
  • Codex 实验的规模远小于其他智能体——例如只在一个小数据集、一个模型上做实验(见案例 3)。这降低了其幻觉和捏造率,但由于实证评估过于有限,整体分数也随之偏低。
  • Kimi Code 则截然不同,它经常直接捏造实验结果,而不是真正运行实验(见案例 4)。即使输出了代码,所提方法与实现之间也常存在不一致,实验失败率也较高(见案例 5)。

更宽泛地看,所有智能体都倾向于与较旧的方法而非最新方法进行比较,即使新近基线已在相关工作章节中被提及。

评审者发现了什么(以及漏掉了什么)

在评审方面,我们发现 SAR 和同行评审(PR)都倾向于对负结果持正面态度——常将其视为诚实与优点,而非缺点。在新颖性评估上,SAR 在检索相关工作方面明显弱于 PR,从而一贯给出更高的新颖性分数。由于 SAR 无法检查代码产物,它几乎从不质疑实验结果的有效性,而主要检查论文不同部分的论断与实验部分报告之间是否存在不一致。

在 PR 中,Codex 识别出的捏造结果最多,Claude Code 其次,Kimi Code 最少。我们人工复核后发现 Codex 和 Claude 提出的绝大多数问题是有效的;相比之下,Kimi Code 的诚信检查最为肤浅,偶尔还会出现误报。整体上,由于无法检查代码产物,SAR 相对 PR 显著高估了自动研究论文的质量,这凸显代码产物检查应当成为未来智能体评审系统的核心功能之一。最后,大多数引用问题并非完全虚构的参考文献,而是作者姓名或会议出处的错误——被引用的论文本身通常是真实存在的。

这带我们走到哪里

总体而言,就论文质量而言,当前所有智能体都远未达到顶级会议的门槛。尽管如此,一些论文确实提供了有价值的洞见,尤其是在较强的案例中,如案例 1。一个核心挑战仍然是如何有效地让智能体设计并执行完整、严谨的实验。同时,我们的发现也引出了更广泛的担忧——当前前沿模型的忠实性。
